# docs

dcc support プロジェクトのドキュメント全体マップ。

## ドキュメントの正と参照順

| 種類 | 場所 | 役割 |
| ------ | ------ | ------ |
| 仕様・ロードマップ | `docs/roadmap/` | プロジェクト仕様書・開発フェーズ定義 |
| 検討ログ | `docs/memory/` | 開発着手に至るまでの思考の履歴 |
| コーディング規約 | `.claude/rules/` | コーディング時の留意事項（Claude Code ルールとして管理、毎セッション自動読込） |
| スキル | `.claude/skills/` | タスク手順書・ワークフロー（`/name` で呼出時のみ読込） |

## プロジェクトに着手する場合

1. **`memory/README.md`** — プロダクトの思想を言語化した経緯
2. **`roadmap/loadmap.md`** — プロジェクト仕様・技術スタック・開発フェーズ

## 用語集

| 用語 | 意味 |
| ------ | ------ |
| coach | AIサブエージェント。画面を見て方針判断・進捗評価・GUI操作指示を行う |
| researcher | AIサブエージェント。coachが確信を持てない操作手順をWeb検索で調査する |
| スキルファイル | アプリ固有のGUI知識ベース（メニュー構造・パネル位置・ショートカット）。システムプロンプトに注入してcoachの精度を上げる |
| プラン | リファレンスと目標を基にAIが生成する制作手順のステップ分解 |
| コーチングループ | 5秒間隔でスクリーンショットを取得し、差分検知→AI呼び出しを繰り返す自律監視ループ |
| 差分検知 | pixelmatchによる前回画像との変化率算出。閾値を超えた場合のみAIを呼ぶトリガー |
| プログレスダッシュボード | ブラウザで開く簡易UI。プランの進捗とアドバイス履歴を表示する |

## 注意事項

- `docs/memory/` の内容は「すでに通過した思考」であり、再検討・再提案の対象ではない
- memory の書き換えは原則禁止（過去の思考そのものは事実であり、改竄すべきではない）

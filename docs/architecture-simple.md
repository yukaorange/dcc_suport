## このプロジェクトは何？

Adobe CC（Photoshop / Illustrator 等）で制作中、**AIが画面を見てリアルタイムでコーチングする**ローカルツール。

```text
あなた: Photoshopで作業中
  ↓ 5秒ごとにスクリーンショットを取得
AI: 「いい感じ。次はオーバーレイで色を重ねてみては」
  ↓ ダッシュボードにアドバイスが表示される
```

## 4つのパッケージ（モノレポ構成）

```text
packages/
├── core/      共有ロジック（AI呼び出し、画面キャプチャ、プラン生成）
├── server/    Webサーバー + API + データベース
├── client/    ブラウザUI（React）
└── cli/       コマンドライン版（既存機能の維持用）
```

| パッケージ | 一言で | 使う技術 |
|-----------|-------|---------|
| **@dcc/core** | 頭脳。AIとの対話やプラン生成 | Claude Agent SDK, pixelmatch, sharp |
| **@dcc/server** | 配膳役。UIとcoreの橋渡し | Hono, tRPC, bun:sqlite |
| **@dcc/client** | 見た目。ブラウザで操作するUI | React, Vite, TanStack Query |
| **@dcc/cli** | 旧版。ターミナルで操作するUI | inquirer |

### 依存の方向

```text
cli ────► core ◄──── server ◄···· client（型だけ参照）
```

core は誰にも依存しない。server と cli が core を使う。client は server の型だけ見る。

## データの流れ

### セットアップ → コーチング開始

```text
1. ブラウザ: ディスプレイ選択 + 画像アップロード + 目標入力
      │
      │ tRPC mutation (plan.generate)
      ▼
2. サーバー: AIにプランを生成させる → メモリに一時保存
      │
      │ tRPC mutation (setup.start)
      ▼
3. サーバー: DB保存 → コーチングループ開始
```

### コーチング中（5秒ごと）

```text
4. サーバー: 画面キャプチャ → 差分チェック → AIに質問
      │
      │ onEvent（アドバイス発生）
      ▼
5. EventBus: サーバー内部で通知を配る
      ├──► SQLiteに保存
      └──► SSEでブラウザに配信
              │
              ▼
6. ブラウザ: ダッシュボードにアドバイスが表示される
```

## 起動コマンド

```bash
# GUI版（ブラウザでダッシュボード）
bun run start:web

# CLI版（ターミナルで操作・既存機能）
bun run start

# 開発（サーバー + Vite を同時起動）
bun run dev:web
```

## 用語早見表

| 用語 | 意味 |
|------|------|
| Hono | Webサーバーのフレームワーク。Express の軽量版 |
| tRPC | バックエンドの関数をフロントから型付きで呼べる仕組み |
| SSE | サーバーからブラウザへの一方向リアルタイム通信 |
| EventBus | サーバー内部で「何かが起きた」を通知する仕組み |
| bun:sqlite | Bunに組み込みのローカルDB。ファイル1つで動く |
| WAL | SQLiteの高速モード。読み書きが同時にできる |
| Vite | フロントエンド開発サーバー。変更を即座にブラウザに反映 |
| TanStack Query | サーバーデータの取得・キャッシュを管理するReactライブラリ |

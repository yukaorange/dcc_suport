# S字カーブ閾値オフセットによるエッジ抽出

ノイズパターンの境界線（エッジバンド）だけを抽出する汎用技法。ディゾルブ演出のエッジ発光などに使う。

## 核心の考え方

### S字カーブ = 閾値関数（Step関数）の近似

Curves の S字カーブは、入力値を「白か黒か」に振り分ける**閾値関数**として機能する。

- **緩やかなS字**: 中間グレーが残る（閾値の遷移領域が広い）
- **急峻なS字**: ほぼ完全な二値（Step関数に近い）

S字の「中間点」が閾値の位置を決める。中間点より上の入力は白へ、下の入力は黒へ振り分けられる。

### 2つのS字の閾値オフセット = エッジバンド

同じノイズ入力に対して、**閾値位置が微妙に異なる2つのS字カーブ**を適用し、減算（Subtract）で差分を取る。

```
ノイズの輝度値 →  0 ----[A閾値]--[B閾値]---- 255

A の出力:  閾値Aより上 → 白、閾値Aより下 → 黒  （白の面積が広い）
B の出力:  閾値Bより上 → 白、閾値Bより下 → 黒  （白の面積が狭い）

A - B（Subtract）:
  - 両方白（閾値Bより上）:  255 - 255 = 0  → 黒
  - 両方黒（閾値Aより下）:  0 - 0 = 0      → 黒
  - Aだけ白（A閾値〜B閾値の間）: 255 - 0 = 255 → 白 ← これがエッジバンド
```

**差分に残るのは、2つの閾値に挟まれた帯域のピクセルだけ**。これがノイズパターンの境界線に沿ったエッジバンドとなる。

## なぜ両方ともS字でなければならないか

### 成功: S字 vs S字（閾値オフセット）

```
A: ──────╱╱──────  （閾値 X で二値化）
B: ────────╱╱────  （閾値 X+δ で二値化）
A-B:      ▌▌        （δ の幅のエッジバンドだけが残る）
```

両方が同じ急峻さで二値化 → 差分は閾値ラインの周辺の薄い帯に限定。

### 失敗: S字 vs 直線

```
A: ──────╱╱──────  （S字で二値化）
B: ────────╱─────  （直線 = 中間グレーが大量に残る）
A-B: ▓▓▓▓▓▓▓▓▓▓▓  （面的な差分がノイズとして広がる）
```

B が中間グレーを保持 → A（二値化済み）との差が**広い面積に散らばる**。エッジではなく面的ノイズになる。

### 直感的な理解

S字カーブは「このピクセルは白か？黒か？」を判定する**投票**のようなもの。AとBが同じルール（同じS字の形）で投票し、閾値位置だけが微妙に違えば、**意見が割れるのは境界付近のピクセルだけ**。一方が投票せず中間のまま（直線）だと、AとBの「意見の違い」が全面に広がってしまう。

## なぜ Brightness オフセットではなく Curves オフセットか

### Brightness オフセット方式（非推奨）

```
A: フラクタルノイズ(Brightness +30) → S字カーブ → 出力
B: フラクタルノイズ(Brightness  0 ) → S字カーブ → 出力
```

Brightness は全ピクセルに一律加算。理論上は「S字カーブへの入力がずれる ≒ 閾値がずれる」のと同等。しかし実際には：

- S字カーブが完全なStep関数でない限り、勾配部分でA・Bの中間値が異なる
- その差が面的なグレーとしてSubtract結果に漏れ出す
- Contrast × S字の急峻さ × Brightnessオフセットの3変数の調整が難しい

### Curves 閾値オフセット方式（推奨）

```
A: フラクタルノイズ(同一設定) → S字カーブ(閾値 X)   → 出力
B: フラクタルノイズ(同一設定) → S字カーブ(閾値 X+δ) → 出力
```

- AとBのフラクタルノイズ出力が**完全に同一**（同じ Contrast、Brightness、Random Seed）
- 違いは**Curvesの閾値位置だけ**
- 同一入力に対して閾値だけが異なるので、差分は**純粋に閾値間の帯域のピクセルだけ**

### AEのSubtractブレンドモードの仕様

- `A - B` で計算され、結果が負になった場合は **0にクランプ**（黒になる）
- 絶対値にはならない。マイナスが白く反転することはない
- つまり**A > B の場所だけ**が白く残る

## エッジ幅の制御

| パラメータ | エッジへの影響 |
|-----------|-------------|
| A と B のS字中間点の差（δ） | **大きい → 太いエッジ**、小さい → 細いエッジ |
| S字カーブの急峻さ | **急峻 → シャープなエッジ**、緩やか → ぼけたエッジ |
| フラクタルノイズの Contrast | 高いほど白黒の遷移勾配が急 → エッジが細くなる |
| Glow Radius（後処理） | エッジ自体の幅とは無関係に、発光の広がりを制御 |

### 調整の優先順位

1. **まずS字の形を決める** — 十分に急峻にして二値化の品質を確保
2. **次にS字の中間点をずらしてエッジ幅を調整** — Bの中間点を右にドラッグ
3. **最後にGlow Radiusで見た目の発光幅を調整** — エッジ自体は細くてOK、グローで広げる

## 実装手順（AE の場合）

1. **フラクタルノイズ + Curves** のレイヤーを作成（A）
   - Fractal Noise: Contrast 200 程度
   - Curves: 急峻なS字カーブ（中間グレーを潰す）
2. レイヤーを **複製** して B とする（Random Seed が自動的に同一になる）
3. **B の Curves の中間点だけを右にずらす**（白になる範囲を A より狭くする）
   - Fractal Noise の設定（Contrast, Brightness, Random Seed 等）は A と完全に同一のまま
4. B のブレンドモードを **減算（Subtract）** に設定
5. 結果 = エッジバンド（黒背景に白い境界線）

## 後処理

エッジバンドが得られたら、以下で発光エフェクトに仕上げる：

- **ガウスぼかし（Gaussian Blur）**: エッジを滑らかにする
- **色合い（Tint）/ 三色調（Tritone）**: エッジにカラーを付ける
- **グロー（Glow）/ Deep Glow**: 発光を追加
- 描画モード: **加算（Add）** または **スクリーン（Screen）**

## Unity Shader Graph との対応

| Unity Shader Graph | AE での対応 |
|---|---|
| Noise テクスチャ | フラクタルノイズ（Fractal Noise） |
| Step ノード | Curves の急峻なS字カーブ |
| Step の閾値 | S字カーブの中間点の位置 |
| 閾値のオフセット | AとBのS字中間点の差 |
| Subtract | 減算ブレンドモード（Subtract） |
| Emission × エッジ | Tint / Glow でカラーと発光を付加 |

## 応用

この技法の本質は「同じ閾値関数の閾値位置をオフセットして差分を取る」こと。以下に応用可能：

- **ディゾルブエッジ発光**: Brightness をアニメーションして閾値ラインを時間的に移動 → エッジも追従
- **炎・溶岩の輪郭**: ノイズ境界に赤〜オレンジのグローを適用
- **魔法陣・エネルギー線**: 低周波ノイズのエッジバンドに加算合成で発光色を乗せる
- **ホログラム境界**: エッジバンドに色収差やグリッチを加える
- **輪郭抽出**: 任意のグラデーション画像に対して、S字の閾値オフセット差分でエッジを抽出
